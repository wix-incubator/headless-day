# AGENTS.md — Exploded Horology Lab (Wix Headless template)

A Claude Design single-page watch atelier, pre-wired for **Wix Headless** (`operation: connect`,
`frontendBuild: none` — static HTML, no build step). This file documents the architecture and how an
agent (or a human) connects it to a live Wix backend. **No site-specific IDs are committed** — fill
your own in `index.html` and `wix.config.json` (git-ignored; generated by `init`).

## What the site is
- A Three.js **exploded-view 3D lab** (`watch-lab.js`) — scroll to disassemble the movement; click a
  part to inspect it. Only the crown's grip winds (back-and-forth); the case and strap don't spin.
- A **configurator** — bezel/dial/strap/crown/movement with running total from a base price.
- A **reserve → checkout** flow (see below).
- Runs on the Claude Design runtime (`support.js`); served as static files.

## Reserve + checkout flow (injected `<script type="module">` at the end of `index.html`)
On the modal's **"RESERVE & PAY"** button, a capture-phase handler:
1. Records the build to a **`Reservations` Wix CMS collection** via `@wix/data` — columns:
   `title` (commission №), `fullName`, `email`, `bezel`, `dial`, `strap`, `crown`, `movement`,
   `total`, `downpaymentPaid` (= base charged now), `outstanding`/`delta` (upgrades, arranged
   separately), `status`.
2. Redirects to a **real Wix hosted checkout** for the **base** amount: `@wix/ecom`
   `currentCart.addToCurrentCart` → `createCheckoutFromCurrentCart` → `@wix/redirects`
   `createRedirectSession` → `location.assign(fullUrl)`.

**Model:** checkout always charges the base; any configured **delta** is recorded in the CMS and
arranged with the client separately. The modal explains this before payment (delta → "we'll contact
you for the balance"; base → "we'll notify you when it's ready").

**The wiring is inert** (`WIX_CONNECTED` guard) until you replace `YOUR_WIX_APP_ID` /
`YOUR_PRODUCT_ID` / `YOUR_VARIANT_ID` at the top of that script — until then the design's own
front-end "Reserved" confirmation runs.

### Load-bearing details (don't regress)
- **Cart line item MUST include the variant** when the product has options:
  `catalogReference: { appId: "215238eb-22a5-4c36-9e7b-e7c08025e04e" /* Wix Stores, constant */, catalogItemId: PRODUCT_ID, options: { variantId: VARIANT_ID } }`.
  Without `options.variantId` the checkout is created EMPTY (0 line items, $0).
- The product must be **purchasable** — if it has no inventory item it reads OUT_OF_STOCK. For
  made-to-order, create an inventory item with `trackQuantity:false, inStock:true`.
- `npx @wix/cli token` intermittently returns an EMPTY string → always mint with a non-empty retry
  loop before use; an empty token yields a misleading `403`.
- SEO tags live in the real static `<head>` (not the DC `<helmet>`), so crawlers get them without
  running JS; the DC runtime preserves them in the rendered DOM too.

## Setup (REST, per capability)
Auth for every call: `Authorization: Bearer $(npx @wix/cli@latest token --site "$SITE_ID")` +
`wix-site-id: $SITE_ID`.

- **Install an app:** `POST https://www.wixapis.com/apps-installer-service/v1/app-instance/install`
  `{"tenant":{"tenantType":"SITE","id":"<siteId>"},"appInstance":{"appDefId":"<appDefId>","enabled":true}}`
  - Wix Stores `215238eb-22a5-4c36-9e7b-e7c08025e04e` (pulls in eCommerce) · Wix Forms
    `225dd912-7dea-4738-8688-4b8c6955ffc2` · Wix CMS/Data `e593b0bd-b783-45b8-97c2-873d42aacaf4`.
- **Create the CMS collection:** `POST https://www.wixapis.com/wix-data/v2/collections` with the
  columns above and `permissions: { insert: "ANYONE", read: "ADMIN", update: "ADMIN", remove: "ADMIN" }`.
- **Create the product:** `POST https://www.wixapis.com/stores/v3/products` (base price + option groups),
  then create an inventory item (above) so it's purchasable.

## SEO / GEO
- On-page `<head>`: title, meta description, canonical, theme-color, inline SVG favicon, Open Graph +
  Twitter, and JSON-LD (`Organization` + `Product`). Replace `https://your-site.example` with your URL.
- `og.jpg` (1200×630) is served at `/og.jpg` (keep it in the project root — it's part of the deployable).
- **robots.txt / llms.txt are NOT static files** — Wix intercepts those root paths. Set them via:
  - `PUT https://www.wixapis.com/promote-seo-robots-server/v2/robots` `{"robotsTxt":{"content":"…","default":false,"subdomain":"www"}}`
  - `PUT https://www.wixapis.com/promote-seo-robots-server/v2/llms`   `{"llmsTxt":{"content":"…","default":false,"subdomain":"www"}}`
  - For llms: Wix prepends an H1 from the business name and wraps line 1 as a `>` tagline, then
    appends AI-Agent/MCP sections — start custom content with the tagline, not an H1. Don't use
    `default:true` for llms (observed to 404).

## Redeploy
Static, no build: `npx @wix/cli@latest release` (publishes `wix.config.json.site.outputDirectory`,
set to `"."`). `index.html` must stay the entry (Wix serves `index.html` at the site root).
